## 5. 통합 테스트 시나리오

### IT-01: 고객 주문 전체 플로우
```
테이블 로그인 → 메뉴 조회 → 장바구니 추가 → 주문 확정 
→ 주문 성공(5초 리다이렉트) → 인구통계 수집 → 별점 평가 
→ 주문 내역 확인 → 관리자 대시보드에 실시간 표시
```

| 단계 | 확인 사항 |
|---|---|
| 1. 테이블 로그인 | JWT 발급, 세션 생성 |
| 2. 메뉴 조회 | 카테고리별 메뉴 표시, 이미지/별점 포함 |
| 3. 장바구니 | 추가/수량 변경/총액 계산 정상 |
| 4. 주문 확정 | API 호출 성공, 주문 번호 반환 |
| 5. 주문 성공 | 5초 후 메뉴 화면 리다이렉트, 장바구니 클리어 |
| 6. 인구통계 | 성별/나이대 저장 |
| 7. 별점 평가 | 메뉴별 별점 저장 |
| 8. SSE 알림 | 관리자 대시보드에 2초 이내 주문 표시 |

### IT-02: 관리자 주문 관리 플로우
```
관리자 로그인 → 대시보드 조회 → SSE 연결 → 주문 상태 변경 
→ 주문 삭제 → 이용 완료 → 과거 내역 확인
```

| 단계 | 확인 사항 |
|---|---|
| 1. 관리자 로그인 | JWT 발급 (16시간) |
| 2. 대시보드 | 테이블별 그리드 표시 |
| 3. SSE 연결 | EventSource 연결 성공 |
| 4. 상태 변경 | 대기중 → 준비중 → 완료 |
| 5. 주문 삭제 | 삭제 후 총액 재계산, SSE 이벤트 |
| 6. 이용 완료 | 주문 아카이브, 테이블 리셋 |
| 7. 과거 내역 | 아카이브된 주문 조회 가능 |

### IT-03: 메뉴 관리 → 고객 반영 플로우
```
관리자 메뉴 등록(이미지 포함) → 고객 메뉴 화면에서 새 메뉴 확인
→ 메뉴 수정 → 고객 화면 반영 → 메뉴 삭제 → 고객 화면에서 제거
```

### IT-04: 추천 시스템 플로우
```
다수 고객 주문 + 인구통계 수집 → 동일 성별/나이대 추천 모달 
→ 인기 메뉴 추천 정확성 확인 → 데이터 부족 시 전체 인기 폴백
```
