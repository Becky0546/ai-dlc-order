# 테이블오더 서비스 - 요구사항 정의서

## Intent Analysis

- **User Request**: 테이블오더 서비스 구축 (디지털 주문 시스템)
- **Request Type**: New Project (Greenfield)
- **Scope**: System-wide (고객 UI + 관리자 UI + 백엔드 서버 + DB)
- **Complexity**: Complex (실시간 SSE, 인증, 추천 시스템, 평가 시스템, S3 이미지 관리)
- **Depth**: Comprehensive

---

## 1. 기술 스택

| 영역 | 기술 |
|---|---|
| **백엔드** | Java + Spring Boot |
| **프론트엔드** | React (TypeScript) |
| **데이터베이스** | MySQL/MariaDB |
| **이미지 저장소** | AWS S3 |
| **배포 환경** | AWS ECS |
| **실시간 통신** | Server-Sent Events (SSE) — 관리자 주문 모니터링용 |

---

## 2. 아키텍처 결정 사항

- **배포 구조**: 단일 웹 앱에서 경로(route)로 고객/관리자 분리 (예: `/customer`, `/admin`)
- **매장 관리 모델**: 단일 매장 시스템 — 하나의 서버 인스턴스가 하나의 매장만 서비스
- **규모**: 중규모 (5~50개 매장, 각 매장별 독립 인스턴스 배포)

---

## 3. 기능 요구사항

### 3.1 고객용 기능 (Customer Features)

#### FR-C01: 테이블 태블릿 자동 로그인 및 세션 관리
- **초기 설정 (관리자 1회 수행)**: 매장 식별자, 테이블 번호, 테이블 비밀번호 입력
- 최종 로그인 정보 로컬 저장 (localStorage/cookie)
- 1회 로그인 성공 후, 저장된 정보로 자동 로그인

#### FR-C02: 메뉴 조회 및 탐색
- **기본 화면**: 메뉴 화면이 기본 화면으로 항상 표시
- 카테고리별 메뉴 분류 및 표시
- 메뉴 상세 정보: 메뉴명, 가격, 설명, 이미지 (S3 URL)
- 카테고리 간 빠른 이동
- **UI**: 카드 형태 레이아웃, 터치 친화적 버튼 (최소 44x44px)

#### FR-C03: 장바구니 관리
- 메뉴 추가/삭제, 수량 조절 (증가/감소)
- 총 금액 실시간 계산
- 장바구니 비우기
- **로컬 저장**: 페이지 새로고침 시에도 유지 (localStorage)
- 서버 전송은 주문 확정 시에만 수행

#### FR-C04: 주문 생성
- 주문 내역 최종 확인 화면
- 주문 확정 버튼
- **주문 성공 플로우**: 주문 번호 표시 → 인구통계 수집 모달(FR-C06, 즉시) → 5초 후 메뉴 화면 자동 리다이렉트 → 장바구니 자동 비우기 → 1분 후 별점 평가 모달(FR-C07)
- 주문 실패 시 에러 메시지 표시 및 장바구니 유지
- **주문 정보**: 매장 ID, 테이블 ID, 주문 메뉴 목록 (메뉴명, 수량, 단가), 총 주문 금액, 세션 ID

#### FR-C05: 주문 내역 조회
- 주문 시간 순 정렬
- 주문별 상세: 주문 번호, 주문 시각, 주문 메뉴 및 수량, 주문 금액
- **현재 테이블 세션 주문만 표시** (이전 식사 세션 제외)
- 매장 이용 완료 처리된 주문은 제외
- 페이지네이션 또는 무한 스크롤
- **참고**: 고객 측 주문 상태 실시간 업데이트 기능은 스펙에서 제거됨

#### FR-C06: 성별/나이대 기반 메뉴 추천
- **데이터 수집**: 주문 완료 후 고객에게 성별(남/여)과 나이대(10대/20대/30대/40대/50대 이상) 선택 화면 표시
- 선택된 인구통계 정보를 해당 주문 데이터와 연결하여 저장
- **추천 표시**: 메뉴 화면 진입 후 1분 뒤 자동 모달 팝업
- 모달에서 성별/나이대 선택 → 동일 인구통계 고객들의 인기 메뉴 추천
- **추천 알고리즘**: 주문 데이터 기반 — 동일 성별/나이대 고객들이 많이 주문한 메뉴를 자동 추천
- 추천 데이터가 충분하지 않을 경우 전체 인기 메뉴 표시

#### FR-C07: 메뉴별 별점 평가
- **주문 완료 후 1분 뒤** 별점 평가 모달 자동 표시
- 각 메뉴에 대해 별점 평가 (1~5점)
- 평가는 선택사항 (건너뛰기 가능)
- 평가 결과는 메뉴 상세 정보에 평균 별점으로 표시

### 3.2 관리자용 기능 (Admin Features)

#### FR-A01: 매장 인증
- 매장 식별자 + 사용자명 + 비밀번호 입력
- **세션**: JWT 토큰 기반, 16시간 유지, 브라우저 새로고침 시 세션 유지, 16시간 후 자동 로그아웃
- **보안**: bcrypt 해싱, 로그인 시도 제한

#### FR-A02: 실시간 주문 모니터링
- **SSE 기반 실시간 업데이트** (2초 이내 주문 표시)
- **그리드/대시보드 레이아웃**: 테이블별 카드 형태 표시
- 각 테이블 카드: 총 주문액, 최신 주문 n개 미리보기
- 주문별 상세: 테이블 번호, 주문 번호, 시각, 메뉴/수량 (축약), 총 금액
- 주문 카드 클릭 시 전체 메뉴 목록 상세 보기
- 주문 상태 변경 (대기중/준비중/완료)
- 신규 주문 시각적 강조 (색상 변경, 애니메이션)
- 테이블별 필터링 기능

#### FR-A03: 테이블 관리
**1. 테이블 태블릿 초기 설정:**
- 테이블 번호 및 비밀번호 설정
- 16시간 세션 생성
- 설정 정보 저장 및 자동 로그인 활성화

**2. 주문 삭제 (직권 수정):**
- 특정 주문 삭제 버튼 → 확인 팝업 → 즉시 삭제
- 테이블 총 주문액 재계산
- 성공/실패 피드백

**3. 테이블 세션 처리:**
- 세션 시작 (첫 주문 시) 및 종료 관리 (이용 완료)
- 이용 완료 시: 주문 내역을 과거 이력(OrderHistory)으로 이동, 현재 주문 목록 및 총 주문액 0으로 리셋
- 확인 팝업 및 성공/실패 피드백

**4. 과거 주문 내역 조회:**
- 테이블별 과거 주문 목록 (시간 역순)
- 주문 정보: 주문 번호, 시각, 메뉴 목록, 총 금액, 이용 완료 시각
- 날짜 필터링 기능

#### FR-A04: 메뉴 관리 (CRUD)
- 메뉴 조회 (카테고리별)
- 메뉴 등록: 메뉴명, 가격, 설명, 카테고리, 이미지 (S3 업로드)
- 메뉴 수정 / 삭제
- 메뉴 노출 순서 조정
- **검증**: 필수 필드, 가격 범위

## 4. 비기능 요구사항

### NFR-01: 성능
- SSE 기반 주문 알림: 2초 이내 관리자 화면 표시
- 매장당 동시 테이블 수: 최대 50개 테이블 지원

### NFR-02: 가용성
- AWS ECS 기반 컨테이너 배포
- 매장별 독립 인스턴스 (단일 매장 시스템)

### NFR-03: 데이터 관리
- 주문 이력 데이터베이스 저장 (OrderHistory 테이블)
- 세션 ID 기반 주문 그룹화
- 인구통계별 주문 데이터 저장 (추천 시스템용)
- 메뉴 이미지: AWS S3 저장 및 URL 참조

---

## 5. 제외 사항

- 결제/PG 연동, 영수증, 환불
- OAuth/SNS/2FA 인증
- 푸시/SMS/이메일 알림
- 주방 전달 및 재고 관리
- 매출 리포트, 예약, 리뷰 시스템, 다국어
- 외부 플랫폼 연동 (배달, POS 등)
- 이미지 리사이징/최적화
- 고객 측 주문 상태 실시간 업데이트
- 1km 반경 모아서 주문하기 기능
